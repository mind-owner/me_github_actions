# GitHub Actions 示例集合

本目录包含各种 GitHub Actions 示例，按功能分类组织，帮助开发者快速学习和使用 GitHub Actions。

## 目录结构

### 1. ci-cd/ - CI/CD 相关示例

包含持续集成和持续部署相关的工作流示例，如：
- 多语言项目构建和测试
- 代码质量检查
- 自动版本控制

### 2. deployment/ - 部署相关示例

包含各种部署场景的工作流示例，如：
- 部署到 GitHub Pages
- 部署到云服务（AWS、Azure、GCP）
- Docker 镜像构建和推送

### 3. automation/ - 自动化任务示例

包含各种自动化任务的工作流示例，如：
- 定时任务
- 代码自动更新
- 依赖自动升级
- 文档自动生成

### 4. notification/ - 通知相关示例

包含各种通知集成的工作流示例，如：
- Slack 通知
- Email 通知
- Telegram 通知
- 微信通知

### 5. utilities/ - 实用工具示例

包含各种实用工具和脚本的工作流示例，如：
- 仓库备份
- 数据同步
- 文件处理
- API 调用

## 使用方法

1. 浏览示例目录，找到适合您需求的工作流
2. 复制对应的 YAML 文件到您的项目 `.github/workflows/` 目录
3. 根据您的项目需求修改配置
4. 提交到 GitHub 仓库，触发工作流运行

## 贡献指南

欢迎贡献更多的 GitHub Actions 示例！请遵循以下指南：

1. 在合适的分类目录下创建示例文件
2. 文件名清晰描述示例用途
3. 在文件开头添加详细的注释说明
4. 确保示例可直接使用或易于修改
5. 提交 Pull Request

## 注意事项

- 示例中可能包含敏感信息（如密钥、令牌），请在使用前替换为您自己的信息
- 某些示例可能需要特定的 GitHub 权限或配置
- 请确保您了解每个示例的功能和潜在影响
- 定期更新示例以使用最新版本的 Actions
